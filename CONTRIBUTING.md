### How to Contribute

There are several ways to contribute to this project:

1. **Report bugs** - If you find errors in documentation or examples
2. **Suggest improvements** - Ideas to improve examples or documentation
3. **Add examples** - New components or usage patterns
4. **Improve documentation** - Fix typos, improve explanations
5. **Translate** - Help with translations to other languages

### Contribution Process

1. **Fork the repository**
   ```bash
   # Click "Fork" on GitHub
   git clone https://github.com/YOUR-USERNAME/Roadie.git
   cd Roadie
   ```

2. **Create a branch**
   ```bash
   git checkout -b feature/my-new-feature
   # or
   git checkout -b fix/bug-fix
   ```

3. **Make your changes**
   - Write clear and well-documented code
   - Follow existing code style
   - Add examples if appropriate

4. **Test your changes**
   - Verify examples work
   - Check documentation for typos

5. **Commit and Push**
   ```bash
   git add .
   git commit -m "feat: clear description of change"
   git push origin feature/my-new-feature
   ```

6. **Open a Pull Request**
   - Go to your fork on GitHub
   - Click "New Pull Request"
   - Describe your changes clearly

### Commit Conventions

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Formatting changes (don't affect code)
- `refactor:` - Code refactoring
- `test:` - Add or modify tests
- `chore:` - Maintenance tasks

Examples:
```
feat: add chart component example
fix: correct typo in README
docs: update quick start guide
```

### Code Style

- Use **TypeScript** for code
- Use **Markdown** for documentation
- Indentation: 2 spaces
- Lines: maximum 100 characters (code), 80 characters (docs)
- File names: kebab-case (e.g., `my-component.tsx`)

### What to Contribute

#### Useful Component Examples
- Integration with popular tools
- Data visualizations
- Forms and workflows
- API integrations

#### Needed Documentation
- Step-by-step tutorials
- Common design patterns
- Specific troubleshooting
- Translations to other languages
